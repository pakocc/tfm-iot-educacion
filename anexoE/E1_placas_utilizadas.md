# E.1. Placas utilizadas

Para la integración de sensores y actuadores se emplearon placas **NodeMCU v1.0 (ESP8266)**, seleccionadas por su bajo coste, conectividad Wi‑Fi integrada y compatibilidad con MQTT. Estas placas permiten ejecutar firmware ligero y establecer comunicación directa con el broker Mosquitto desplegado en la Raspberry Pi.

Las placas destacan por:

- [bajo coste y disponibilidad](ca://s?q=NodeMCU_bajo_coste)
- [Wi‑Fi integrado compatible con redes 2.4 GHz](ca://s?q=NodeMCU_wifi_24GHz)
- [soporte nativo para MQTT mediante PubSubClient](ca://s?q=NodeMCU_MQTT_PubSubClient)
- [capacidad para ejecutar firmware ligero en Arduino IDE](ca://s?q=NodeMCU_firmware_Arduino)
- [compatibilidad con sensores y actuadores comunes](ca://s?q=NodeMCU_sensores_actuadores)
